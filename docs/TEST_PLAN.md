# План тестирования интеллектуального сервиса IsNews

## Цель тестирования

Подтвердить, что сервис классификации новостных публикаций корректно загружает данные, обучает модели, сохраняет и загружает артефакты, применяет модели по назначению и формирует результаты оценки качества.

## Объект тестирования

Объектом тестирования является Python-приложение IsNews:

- Streamlit GUI, запускаемый через `main.py`;
- модули загрузки, предобработки и разбиения датасета;
- TF-IDF-векторизация;
- обучение Logistic Regression и MultinomialNB;
- сохранение и загрузка моделей;
- одиночный и пакетный инференс;
- отчеты, графики, таблицы и deployment-материалы;
- Colab/Jupyter notebook-файлы.

## Тестовое окружение

| Параметр | Значение |
| --- | --- |
| ОС | Windows |
| Язык | Python 3.12 |
| Интерфейс | Streamlit |
| Основные библиотеки | pandas, scikit-learn, scipy, joblib, torch, transformers |
| Датасет | `data/raw/news_demo_dataset.csv` |
| Модели | `model.joblib`, `model1.joblib` |

## Методы тестирования

В проекте применяются следующие методы тестирования:

- Функциональное тестирование: проверка загрузки данных, обучения, сохранения, загрузки и применения модели.
- Интеграционное тестирование: проверка связки `data -> preprocessing -> split -> TF-IDF -> model -> evaluation`.
- Регрессионное тестирование: повторный запуск `scripts/verify_project.py` после изменений.
- Smoke-тестирование: быстрая проверка запуска `main.py`, импорта библиотек и загрузки моделей.
- Тестирование данных: проверка обязательных колонок `text` и `label`, количества строк и числа классов.
- Тестирование качества модели: контроль Accuracy на validation и test-выборках.
- Тестирование воспроизводимости: повторное создание датасета и моделей через `scripts/train_demo_artifacts.py`.

## Критерии входа

- Установлены зависимости из `requirements.txt`.
- В корне проекта присутствует `main.py`.
- В проекте есть обучающие данные и сохраненные модели.
- Notebook-файлы находятся в каталоге `notebooks/`.

## Критерии выхода

- Все обязательные файлы проекта присутствуют.
- Python-код компилируется без ошибок.
- Все зависимости импортируются.
- Модели загружаются из файлов.
- Accuracy каждой демонстрационной модели на test-выборке не ниже 0.70.
- Подготовлены тест-кейсы и журнал дефектов.

## Команды проверки

```powershell
python scripts\train_demo_artifacts.py
python scripts\verify_project.py
python main.py
```

## Риски

| Риск | Мера снижения |
| --- | --- |
| Проверяющий запускает проект в пустом окружении | Версии зависимостей закреплены в `requirements.txt`. |
| Модель не совпадает с датасетом | Датасет, split-файлы, признаки и manifest сохранены в репозитории. |
| Метрики невозможно воспроизвести | Добавлен скрипт `scripts/train_demo_artifacts.py`. |
| Изменение библиотеки ломает загрузку модели | Используется закрепление версий и проверочный скрипт. |
