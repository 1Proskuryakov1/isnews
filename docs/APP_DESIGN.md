# Дизайн приложения

## Формат интерфейса

Демонстрационное приложение реализовано как web-интерфейс на Streamlit. Такой формат выбран потому, что проверяющий может запустить сервис локально командой `python main.py` без установки отдельного frontend-сервера.

## Основные пользовательские сценарии

Интерфейс должен поддерживать следующие сценарии:

- Загрузка CSV-датасета из локального файла.
- Загрузка CSV-датасета по прямой ссылке.
- Просмотр сводки по датасету.
- Предобработка текста.
- Разбиение данных на train, validation и test.
- Построение TF-IDF-признаков.
- Обучение Logistic Regression.
- Обучение MultinomialNB.
- Сохранение обученных моделей.
- Загрузка сохраненной модели и векторизатора.
- Одиночная классификация новости.
- Пакетная классификация CSV-файла.
- Оценка качества модели.
- Сравнение нескольких моделей.
- Экспорт отчетов и таблиц для ВКР.

## Логическая структура экрана

Интерфейс строится по этапам ML-пайплайна:

1. Блок загрузки данных.
2. Блок предобработки и анализа датасета.
3. Блок разбиения выборок.
4. Блок векторизации.
5. Блок обучения моделей.
6. Блок оценки качества.
7. Блок загрузки сохраненных артефактов.
8. Блок инференса.
9. Блок отчетов и экспорта.

Такой порядок совпадает с разделами разработки ПО, обучения модели и тестирования модели в ВКР.

## Визуальные элементы

В приложении используются:

- формы загрузки файлов;
- текстовые поля для ввода URL и одиночной новости;
- кнопки запуска этапов обработки;
- таблицы pandas DataFrame;
- метрики Streamlit;
- графики качества и распределений;
- сообщения об ошибках и предупреждениях;
- блоки с путями к сохраненным артефактам.

## Обработка ошибок

Ошибки пользователя должны отображаться понятными сообщениями:

- пустой CSV-файл;
- отсутствие колонок `text` или `label`;
- пустые значения текста или метки;
- недостаточное число строк для разбиения;
- отсутствие сохраненной модели;
- несовместимость модели и векторизатора;
- Accuracy ниже требуемого порога.

## Связь с файлами проекта

Пользовательские действия в GUI приводят к созданию артефактов в стандартных каталогах:

- `data/raw/` - исходные датасеты;
- `data/processed/` - очищенные данные;
- `data/splits/` - train/validation/test выборки;
- `data/features/` - TF-IDF признаки;
- `models/` - модели и векторизаторы;
- `reports/` - отчеты, таблицы, графики и manifest-файлы.
