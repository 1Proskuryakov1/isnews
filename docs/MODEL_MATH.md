# Математическое описание моделей

## Постановка задачи

Пусть имеется набор новостных публикаций:

```text
D = {(x_i, y_i)}_{i=1}^{N}
```

где `x_i` - текст новости, `y_i` - тематическая метка класса. В проекте используются пять классов: политика, экономика, спорт, технологии и культура. Требуется построить функцию классификации:

```text
f(x) -> y
```

которая по тексту новости определяет наиболее вероятную тематическую категорию.

## Предобработка текста

Перед обучением текст приводится к нормализованному виду:

- удаляются лишние пробелы и переносы строк;
- нормализуется регистр;
- удаляются некорректные и пустые строки;
- удаляются дубликаты пар `text` и `label`.

После очистки формируется корпус документов `X = {x_1, ..., x_N}` и вектор меток `Y = {y_1, ..., y_N}`.

## TF-IDF-векторизация

Для преобразования текста в числовые признаки используется TF-IDF. Для термина `t` в документе `d` вычисляется:

```text
tfidf(t, d) = tf(t, d) * idf(t)
```

где `tf(t, d)` - частота термина в документе, а `idf(t)` показывает, насколько термин специфичен для корпуса:

```text
idf(t) = log((1 + N) / (1 + df(t))) + 1
```

`N` - количество документов в обучающей выборке, `df(t)` - количество документов, содержащих термин `t`.

После векторизации каждый текст представляется разреженным вектором:

```text
x_i -> v_i in R^m
```

где `m` - размер словаря TF-IDF-признаков.

## Logistic Regression

Модель Logistic Regression используется как основной классификатор `model.joblib`. Для каждого класса `k` модель вычисляет линейную оценку:

```text
z_k = w_k^T x + b_k
```

где `w_k` - вектор весов класса, `b_k` - свободный коэффициент, `x` - TF-IDF-вектор новости.

Вероятности классов рассчитываются через softmax:

```text
P(y = k | x) = exp(z_k) / sum_j exp(z_j)
```

Итоговый класс выбирается по максимальной вероятности:

```text
f(x) = argmax_k P(y = k | x)
```

Обучение минимизирует функцию логистической потери с регуляризацией:

```text
L = -sum_i log P(y_i | x_i) + lambda * ||W||_2^2
```

Регуляризация снижает переобучение и стабилизирует веса модели.

## Multinomial Naive Bayes

Модель MultinomialNB используется как дополнительная демонстрационная модель `model1.joblib`. Она основана на теореме Байеса:

```text
P(y = k | x) proportional P(y = k) * P(x | y = k)
```

Для текстовой классификации предполагается условная независимость признаков:

```text
P(x | y = k) = product_j P(x_j | y = k)
```

В логарифмической форме классификатор выбирает класс:

```text
f(x) = argmax_k [log P(y = k) + sum_j x_j * log P(t_j | y = k)]
```

Для защиты от нулевых вероятностей применяется сглаживание Лапласа:

```text
P(t_j | y = k) = (count(t_j, k) + alpha) / (sum_l count(t_l, k) + alpha * m)
```

где `alpha` - параметр сглаживания, `m` - число признаков.

## Метрики качества

Основная метрика проекта - Accuracy:

```text
Accuracy = correct_predictions / all_predictions
```

Также рассчитываются Precision, Recall и F1-score:

```text
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 = 2 * Precision * Recall / (Precision + Recall)
```

Для многоклассовой задачи используются macro- и weighted-усреднения. В финальных отчетах сохранены метрики на train, validation и test-выборках.
